# SeaDrop v1.5 Implementation Plan

## Overview
SeaDrop v1.5 requires significant changes from v1.4. Key architectural changes:
- **Windows**: Use TcpListener (inbound) instead of TcpClient (outbound) for main operation
- **Android**: Migrate from AppCompatActivity/XML to Jetpack Compose-only UI
- **Both**: Implement 8-screen setup wizards per specification

---

## Resolved Decisions

1. **TCP Architecture**: Split model - TcpListener (inbound) for ESP32 connections + TcpClient (outbound) only for registration. Registration uses outbound; normal operation uses inbound.

2. **BLE reg_mode**: Add 1-byte reg_mode flag in BLE service data. Format: `[0xAD][0xFE][reg_mode(1 byte)][SSID...][conn_count(1 byte)]`. Windows BleWatcher parses offset 2; Android BleScanner parses the same.

3. **Android fonts**: Add YoungSerif-Regular.ttf and Inter-Regular.ttf to `res/font/` directory with XML font resources.

4. **Android Compose**: Add Compose BOM dependencies (material3, runtime, ui) and migrate RegistrationActivity to Composables.

5. **Firmware NCSI**: Already implemented in `ncsi.cpp` with DNS spoof and HTTP handlers for Windows/Android probes.

## Phase 3: Firmware Update (BLE reg_mode)

### 3.1 Modify `ble_server.cpp`
- Change service_data format:
```cpp
// OLD: [UUID][SSID][conn_count]
// NEW: [UUID][reg_mode][SSID][conn_count]
service_data[0] = 0xAD;  // UUID low byte
service_data[1] = 0xFE;  // UUID high byte
service_data[2] = (uint8_t)g_reg_state.active;  // reg_mode
memcpy(service_data + 3, ssid, ssid_len);
service_data[3 + ssid_len] = (uint8_t)conn_count;
fields.svc_data_uuid16_len = 3 + ssid_len + 1;
```

### 3.2 Update `protocol.hpp` (optional)
- No changes needed - reg_mode is internal state flag
